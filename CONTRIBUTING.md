# Contributing to CircuitCore

One monorepo. Multiple tools sharing a canonical PCB model. Discipline
about which code belongs where matters more than usual.

## Where new code goes

| Kind of change | Belongs in |
|---|---|
| Generic PCB model field, hit-test math | `board/` |
| Reading a new PCB file format | `formats/<vendor>/` (one folder per format) |
| Generic utility used by multiple format adapters | top-level (e.g. `sexpr/`) |
| Anything PI-specific (IR drop, plane Z(f), transient) | `pdnkit/` |
| Anything SI-specific (impedance, S-parameters, eye) | `sikit/` |
| Vendored third-party header | `third_party/<org>/` |

**The bar for `board/` and the top-level utilities:** every format
adapter and every analysis tool needs it, and maintaining two copies
would be a bug. If you find yourself wanting to add `board/kicad/` or
`board/ipc2581/`, that is a sign the code belongs in `formats/`, not in
the canonical model.

## Build & test

```
CC=clang CXX=clang++ cmake -B build -G Ninja
cmake --build build
ctest --test-dir build
```

Tests are organized per library: `sexpr/tests/`, `formats/kicad/tests/`,
`pdnkit/tests/`, `sikit/tests/`. Filter with ctest's `-R` flag to run
just one slice.

## Adding a new PCB format adapter

1. Create `formats/<vendor>/` with `include/circuitcore/formats/<vendor>/`,
   `src/`, `tests/`, and a `CMakeLists.txt` modeled after
   `formats/kicad/CMakeLists.txt`.
2. The adapter's public API is **one function**: take a file path,
   return a `circuitcore::board::Board`. Throw a typed exception on
   parse error.
3. Add `add_subdirectory(formats/<vendor>)` in the top-level
   CMakeLists.txt.
4. Tests must parse a small fixture and assert the board content
   matches expectations. **Do not** test analysis behavior in
   `formats/<vendor>/tests/` -- that belongs in the tool's test
   directory.

## Cross-tool refactors

If a `board/` change requires updating both pdnkit and sikit, make it
**one commit**. That is the whole point of the monorepo -- atomic
refactors across boundaries.

## Commits

- Subject line: imperative, <=72 chars.
- Body: why and any tradeoffs, not just what. Include numerical
  impact for physics changes.
- No Co-Authored-By Claude trailers or autogenerated lines.
- Use the GitHub noreply email
  (`167274875+UnsignedChad@users.noreply.github.com`) so commits link
  to your profile.

## Style

- C++23 (requires clang ≥ 17 or gcc ≥ 13), `-Wall -Wextra -Wpedantic`.
- `clang-format` is configured; run it before committing.
- Headers: `#pragma once`. PIMPL only when there is a real ABI reason.
- Types in `PascalCase`, functions in `snake_case`, members with
  trailing underscore. Namespaces lowercase
  (`circuitcore::board`, `circuitcore::sexpr`,
  `circuitcore::formats::kicad`, `pdnkit::pi`, `sikit::si`).
